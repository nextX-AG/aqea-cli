# Contributing to AQEA CLI

Thank you for your interest in contributing to AQEA CLI!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs
- Include your OS, architecture, and CLI version
- Provide steps to reproduce the issue

### Feature Requests

- Open a GitHub Issue with the `enhancement` label
- Describe the use case and expected behavior

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests (`cargo test`)
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Setup

### Prerequisites

- Rust 1.70+
- Git

### Building

```bash
git clone https://github.com/nextX-AG/aqea-cli.git
cd aqea-cli
cargo build
```

### Running Tests

```bash
cargo test
```

### Code Style

- Run `cargo fmt` before committing
- Run `cargo clippy` and fix warnings

## Code of Conduct

Be respectful and constructive. We're all here to build something great.

## Questions?

Contact us at hello@aqea.ai or open a GitHub Discussion.
